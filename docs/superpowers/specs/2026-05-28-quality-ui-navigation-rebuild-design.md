# 草地监测质检 UI 与导航语义完整重构设计

日期：2026-05-28

## 背景

当前应用已经具备草地监测 ZDB 只读质检主线：选择数据目录、扫描样地、选择范围、执行质检、查看汇总、查看样地详情，并支持问题忽略与取消忽略。现有 Compose UI 与目标稿差距较大，主要问题是视觉系统、页面结构和导航语义都没有贴近 Pencil 与 `stitch_mint_liquid_glass_ui` 的 Liquid Glass 风格。

本次重构以 Pencil MCP 中的 5 个页面为主参考：

- 数据目录选择
- 选择质检范围
- 质检结果汇总
- 质检进行中
- 样地质检详情

HTML 参考目录 `stitch_mint_liquid_glass_ui` 作为尺寸、字号、间距和风格校准来源。Pencil 优先级高于 HTML。

## 目标

1. 完整重做 Compose UI 层和导航语义。
2. 保留业务主线闭环，不重写 ZDB 读取、规则执行、结果复核等核心逻辑。
3. 以 390dp 宽移动端稿为基准，逐屏 1:1 复刻页面结构、控件尺寸、间距、字号层级、颜色和 Liquid Glass 质感。
4. 页面文案使用中文业务语义；字体族可以不完全一致，但字号和层级必须贴近目标稿。
5. 目标稿中存在但当前业务没有的视觉元素也要补齐，例如底部 Dock 的设置入口。
6. 每完成一页后，在询问并获得用户确认后，才进行真机安装、启动、截图和对比。

## 非目标

1. 不重写质检规则引擎。
2. 不改变 ZDB 只读访问原则。
3. 不伪造业务结果。没有真实数据时使用空态或占位态。
4. 不以 mockup 作为验收依据。
5. 不在未询问用户的情况下调用手机或执行 ADB 真机操作。

## 保留的业务主线

以下能力必须保留：

- 选择数据目录并扫描 ZDB
- 选择单样地或全量质检
- 测试模式
- 执行质检和取消质检
- 查看质检汇总
- 进入样地详情
- 忽略和取消忽略问题
- 再次质检
- 返回范围选择

## 可重做范围

以下内容可以彻底重做：

- `QualityScreen` 页面语义
- 页面跳转规则
- 顶栏、底部 Dock、底部浮动操作按钮
- 页面结构和组件层级
- 信息展示顺序
- 公共 UI 组件
- 视觉 token、颜色、圆角、阴影、间距和字号
- 当前没有真实业务的 Dock 入口，用视觉占位或空态承接

## 新导航语义

底部 Dock 统一为四个视觉入口：

- 数据：数据目录选择、扫描状态、扫描结果概览
- 样地：范围选择、搜索、区县筛选、全量模式、测试模式
- 质检：质检进行中、结果汇总、再次质检
- 设置：视觉入口和空态，暂不接入复杂设置业务

详情页作为结果流的二级页面，不一定常驻 Dock，但视觉仍保留目标稿中的顶部栏和底部操作结构。

## 视觉系统

颜色以 Liquid Verdant 参考为准：

- 背景：`#F6FBF6` 与 `#F5F5DC`
- 主色：`#006C51`
- 强调薄荷色：`#8FF6CF`、`#98FFD8`
- 深绿文字：`#1A3A2F`
- 次级文字：`#3E4944`
- 玻璃卡片：半透明白，约 36% 到 60% 不透明度
- 描边：白色半透明细线
- 错误：红色
- 提示：琥珀色
- 忽略：灰绿

关键尺寸：

- 设计基准宽度：390dp
- 顶栏高度：64dp
- 底部 Dock 高度：80dp
- 主内容左右边距：20dp
- 页面主标题：34sp，行高约 41sp
- 顶栏标题：22sp
- 小节标题：18sp
- 正文：16sp 与 14sp
- 标签与 Dock 文案：10sp 到 12sp
- 主卡圆角：12dp
- 主按钮圆角：12dp 或全圆角，根据目标页决定
- 图标按钮：40dp 到 44dp
- 统计格、列表卡、筛选条按 Pencil 尺寸优先

Compose 无原生 backdrop blur 时，使用半透明表面、浅色渐变、白色描边和柔和阴影模拟玻璃质感。动态波纹和 sheen 效果可以用 `Canvas` 或渐变覆盖实现，但不得影响主流程。

## 页面设计

### 数据目录选择

目标结构：

- 64dp 顶栏：菜单、标题、账户图标
- 主标题区：中文标题和说明文案
- 数据目录玻璃卡：圆形目录图标、目录状态、选择目录按钮
- 扫描状态玻璃卡：状态胶囊、扫描数量、索引样地数量、异常数量
- 底部主按钮：进入样地范围选择
- 可选底部 Dock：目标稿无 Dock 时以页面主按钮优先

中文文案保留业务含义，例如“草地监测质检”“数据目录选择”“选择目录”“重新扫描”“下一步：选择范围”。

### 选择质检范围

目标结构：

- 64dp 顶栏
- 大标题与说明
- 筛选玻璃卡：搜索框、区县下拉、全量模式、测试模式
- 可用样地列表：图标、样地编号、区县、选择控件
- 底部浮动主按钮
- 80dp 底部 Dock：样地为选中态，质检、统计、设置为视觉入口

全量模式开启时，列表与筛选控件降权显示，主按钮执行全量质检。

### 质检进行中

目标结构：

- 聚焦任务页，不显示完整 Dock
- 中央圆形 Liquid 进度视觉
- 已完成数量、总数、当前样地
- 取消按钮
- 背景大面积薄荷与暖黄色光晕

保留取消质检逻辑。若业务没有最近完成列表，则不强行伪造。

### 质检结果汇总

目标结构：

- 64dp 顶栏
- 2x2 或响应式统计格：强制性、提示性、已忽略、已跳过
- 测试模式下补充通过和已执行规则
- 样地结果列表：样地编号、区县、问题统计胶囊、进入详情箭头
- 底部双按钮：再次质检、改变范围
- 80dp 底部 Dock：质检为选中态

### 样地质检详情

目标结构：

- 64dp 顶栏：返回、标题、账户图标
- 顶部三格统计：强制性、提示性、已忽略
- 状态筛选条：全部、失败、已忽略
- 表分组筛选
- 问题卡片：严重性标签、规则编号、表名、标题、描述、发现值、忽略或恢复操作
- 底部操作：重新质检、返回结果

问题数据来自现有 `ReviewedPlotResult` 和过滤逻辑。没有的字段用现有规则模型中的可用字段映射，不伪造核心业务结果。

## 真机验证协议

任何真机相关操作前必须先询问用户并获得确认，包括：

- `adb install`
- `adb shell am start`
- `adb shell input`
- `adb shell screencap`
- `adb pull` 真机截图
- 其他会访问已连接手机的命令

每页完成后的验证流程：

1. 本地构建。
2. 询问用户是否允许连接手机进行安装和截图。
3. 用户确认后安装并打开应用。
4. 进入对应页面。
5. 截图并拉回本地。
6. 与 Pencil 截图和 HTML `screen.png` 做人工对比。
7. 记录差异并继续微调。

## 实现策略

1. 先重做视觉底座：
   - `QualityDesignTokens`
   - `QualityAdaptiveLayout`
   - 顶栏、Dock、玻璃卡、按钮、Chip、统计块、问题卡

2. 再重做导航语义：
   - 重新整理 `QualityScreen`
   - 调整页面跳转命名与主线
   - 保持 ViewModel 核心业务调用不变或仅做适配包装

3. 最后逐屏重构：
   - 数据目录选择
   - 选择质检范围
   - 质检进行中
   - 质检结果汇总
   - 样地质检详情

4. 每屏构建后先做本地静态检查，再询问用户是否允许真机验证。

## 风险与处理

- Compose 没有真实 CSS backdrop-filter：用半透明、渐变、描边、阴影模拟。
- 中文文案比英文更长：优先保持目标字号，必要时换行或压缩布局，不随意缩小字号。
- 目标稿存在视觉入口但业务未实现：设置入口先做空态，不新增复杂设置功能。
- 现有工作树有未提交变更：只改与本次 UI 重构相关文件，不回滚用户已有改动。
