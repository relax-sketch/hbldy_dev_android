## Why

草原监测人员目前缺少在 Android 设备上直接对采集项目 `.zdb` 数据进行可解释质检的工具，发现问题后需要依赖人工排查或其他环境处理。首版 App 需要在不修改业务数据库的前提下，将既有规则和少量新增规则转化为现场可反复执行、可定位错误值的检查流程。

## What Changes

- 提供 Android 端数据目录授权与扫描流程，从 `草原监测/数据` 下的项目子目录发现有效 `.zdb` 并构建样地索引。
- 提供单样地、当前区县全部样地和全域全部样地三种检查范围，并支持样地 ID 输入/选择及区县筛选。
- 将当前 ZDB 中的 264 条启用规则提取为 APK 内置基础规则快照，并将其首版统一标记为强制性规则。
- 从草地自然语言规则表中选择 10 条可验证规则转换为内置附加 SQL，保留强制性或提示性标签。
- 对用户 `.zdb` 仅使用只读 SQLite 查询执行规则，返回错误定位字段、实际值、规则解释和结构不兼容跳过原因。
- 提供批量检查进度、按样地汇总结果以及问题详情展示。
- 提供人工忽视/取消忽视与重新质检流程；仍命中的已忽视问题保持绿色状态并排列在问题列表末尾。
- 将忽视标注仅保存在 App 私有本地存储中，不修改或复制用户业务数据。

## Capabilities

### New Capabilities

- `zdb-plot-source-selection`: 授权、扫描和筛选 `.zdb` 数据源中的样地，并定义单样地、区县和全域检查范围。
- `readonly-quality-rule-execution`: 加载 APK 内置规则集并对选定样地集合执行安全的只读 SQL 质检。
- `quality-check-result-review`: 展示批量/单样地检查结果，支持错误解释、人工忽视状态与重复质检。

### Modified Capabilities

无。

## Impact

- 后续实现将影响现有 Android `app` 模块的 Compose 界面、目录读取、SQLite 访问、规则资产加载及 App 私有标注存储。
- 新增 APK 内置规则 JSON 资产和由 JSON 业务样本构建的测试数据准备流程。
- 待检 `.zdb` 为外部只读数据源；不修改其结构、规则表或业务记录。
- 首版不包含规则编辑、结果导出、历史批次对比或后台长期任务。
